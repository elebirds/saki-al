# Prompt 0：总控（先贴给 AI，让它进入“不会跑偏”的模式）

Copilot Prompt：

你在实现一个 Python 项目 saki，一个全栈主动学习框架。目前聚焦于服务是 “Model Runtime”，只负责训练/推理/主动学习选样/产出事件与制品；业务事实（项目/数据集/标注/用户）都在另一个服务 Saki API 中以及对应的操作前端saki-web。

这是一个Python工程，我希望你用uv管理依赖。

硬约束：

Runtime 只能通过 HTTP 调用 Saki 的 internal API 拉取 IR（samples/annotations/labels），不得访问 Saki 数据库。

Runtime 的所有持久化都写在 runs/{job_id}/，包括 config.json、events.jsonl、artifacts/、data/、cache/。

训练/推理必须跑在子进程（subprocess），子进程只用 SDK 往 events.jsonl 写结构化事件；不得直接操作 WebSocket。

API 契约必须强类型（Pydantic），字段名/枚举/状态机必须严格遵守本 prompt（像 gRPC 一样）。

只实现检测任务 MVP：train_detection + query (unit=image, strategy=uncertainty)。

输出要求：写可运行、可维护的生产级代码：类型齐全、错误处理清晰、日志完善、适当注释、包含最小测试或自检脚本。任何你不确定的内容请做合理默认，但不要引入额外依赖与复杂架构（例如 Kafka/Redis/K8s）。我已将AI生成的初步文档放在了MODEL_RUNTIME_DESIGN.md内，但是仅供参考。对于你非常不确定、且API文档有明显设计不合理/冲突，或会造成设计严重后果的前提下，可以向用户询问。

如果你明白了，请先给这个子项目起一个名字，并新建一个相关文件夹，并回复：明白，等待指示。
